# 학생용 평가 루브릭

## 전체 배점

| 평가 요소 | 배점 |
|---|---:|
| 문제 분해 및 문제 모델링 | 10점 |
| 알고리즘 구현 및 제어구조 활용 | 10점 |
| 2차원 리스트와 함수 분리 | 5점 |
| 보고서 작성 | 5점 |
| 합계 | 30점 |

---

## 1. 문제 분해 및 문제 모델링 — 10점

| 점수 | 기준 |
|---:|---|
| 10점 | 주제가 구체적이고 입력·처리·출력 구조가 명확하며, 기능 분해와 데이터 구조, 예외 상황까지 설명함 |
| 8점 | 주제와 입력·처리·출력 구조가 대체로 명확하고 구현 가능한 수준으로 설계함 |
| 6점 | 주제는 있으나 처리 과정이나 데이터 구조 설명이 단순함 |
| 4점 | 주제 선정 또는 계획 작성에는 참여했으나 문제 구조가 명확하지 않음 |
| 0~3점 | 제출물이 없거나 문제 모델링을 확인하기 어려움 |

---

## 2. 알고리즘 구현 및 제어구조 활용 — 10점

| 점수 | 기준 |
|---:|---|
| 10점 | 조건문과 반복문을 복합적으로 활용하고, 검색·필터링·계산·카운팅·정렬·점수 계산·판정 등 의미 있는 알고리즘이 동작함 |
| 8점 | 조건문과 반복문을 활용해 핵심 기능을 구현하고, 입력이나 조건에 따라 결과가 달라짐 |
| 6점 | 일부 기능은 구현했으나 알고리즘이 단순하거나 오류가 일부 있음 |
| 4점 | 단순 출력 중심이거나 핵심 기능이 미완성임 |
| 0~3점 | 코드가 거의 없거나 실행·처리 과정을 확인하기 어려움 |

---

## 3. 2차원 리스트와 함수 분리 — 5점

| 점수 | 기준 |
|---:|---|
| 5점 | 2차원 리스트의 행·열 의미가 명확하고, 기능을 3개 이상 함수로 분리했으며 각 함수 역할을 설명할 수 있음 |
| 4점 | 2차원 리스트와 함수가 사용되었고 기본적인 기능 분리가 이루어짐 |
| 3점 | 2차원 리스트 또는 함수 사용이 형식적이거나 일부만 의미 있게 사용됨 |
| 2점 | 2차원 리스트나 함수 분리를 시도했으나 구조가 불완전함 |
| 0~1점 | 2차원 리스트와 함수 분리 사용을 확인하기 어려움 |

---

## 4. 보고서 작성 — 5점

| 점수 | 기준 |
|---:|---|
| 5점 | 주제, 문제 상황, 입력·처리·출력, 2차원 리스트 구조, 함수 역할, 실행 결과, Gemini 활용, 오류 수정, 개선 방안이 구체적임 |
| 4점 | 주요 항목을 대부분 작성했고 코드와 보고서 내용이 대체로 일치함 |
| 3점 | 보고서는 작성했으나 코드 설명, 실행 결과 분석, AI 활용 기록 중 일부가 부족함 |
| 2점 | 보고서 작성 활동에는 참여했으나 내용이 매우 단순함 |
| 0~1점 | 보고서가 없거나 평가 근거로 보기 어려움 |

---

## 점수 상한이 적용될 수 있는 경우

다음 경우에는 코드가 실행되어도 높은 점수를 받기 어렵습니다.

```text
- 템플릿 예시를 거의 그대로 제출함
- 코드가 실행되지만 핵심 알고리즘을 설명하지 못함
- 2차원 리스트 행·열 의미를 설명하지 못함
- 함수 역할을 설명하지 못함
- Gemini가 만든 코드를 거의 그대로 제출하고 수정·검토 기록이 없음
- 보고서 내용과 실제 코드가 다름
- 코드인터뷰에서 자기 코드를 거의 설명하지 못함
```

핵심 원칙:

> AI는 도구로 사용할 수 있지만, 최종 코드의 이해와 설명 책임은 학생 본인에게 있습니다.
